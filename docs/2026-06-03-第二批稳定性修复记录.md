# 2026-06-03 第二批稳定性修复记录

## 本批目标

在第一批后端稳定性修复的基础上，继续补齐第二批边界与类型安全问题，优先处理：

1. 下架商品详情不应继续对普通用户公开可见。
2. 订单创建事务中的库存竞争应返回明确业务错误，而不是 500。
3. 订单相关前端页面减少 `any` 与旧式 `res.data` 依赖，降低编译期漏检和运行期不确定性。

## 已完成修复

### 1. 商品详情访问边界修复

- 文件：`backend/src/routes/products.ts`
- 调整：
  - 当商品 `isActive=false` 时，普通请求与非管理员请求统一返回 `404 商品不存在`。
  - 若请求头携带有效管理员 Token，则允许查看下架商品详情。
- 收益：
  - 避免前台通过详情接口直接访问已下架商品。
  - 与商品列表的可见性规则保持一致。

### 2. 创建订单库存竞争返回码修复

- 文件：`backend/src/routes/orders.ts`
- 调整：
  - 对事务内 `库存不足` 的业务异常进行单独捕获并返回 `400`。
  - 保留其他未知异常的 `500` 处理。
- 收益：
  - 当并发扣减库存失败时，前端能拿到可展示、可处理的业务消息。
  - 避免把可预期的业务边界误报成系统故障。

### 3. 用户端订单链路类型补强

- 文件：
  - `frontend-user/src/api/orders.ts`
  - `frontend-user/src/pages/Orders.tsx`
  - `frontend-user/src/pages/OrderDetail.tsx`
  - `frontend-user/src/pages/PaymentSuccess.tsx`
  - `frontend-user/src/pages/Checkout.tsx`
- 调整：
  - 为订单状态、订单项、评价、用户摘要、订单实体补充明确 TypeScript 类型。
  - API 层改为直接返回业务数据，而不是把 Axios 响应对象透传到页面。
  - 页面移除订单链路中的 `as any`、`any[]` 和错误消息的非安全读取方式。
- 收益：
  - 查询和变更逻辑的返回结构在编译期即可校验。
  - 减少页面继续误用 `res.data` 的概率。

### 4. 用户端购物车相关联动修复

- 文件：
  - `frontend-user/src/api/cart.ts`
  - `frontend-user/src/components/Layout.tsx`
  - `frontend-user/src/pages/Cart.tsx`
- 调整：
  - 为购物车与商品摘要补充类型。
  - 同步修正购物车接口改为“直接返回数据”后的调用方式。
  - 补齐购物车页面 mutation 的错误类型与变量类型。
- 收益：
  - 避免类型收紧后在购物车角标、购物车页、结算页出现连带编译错误。

### 5. 管理端订单链路类型补强

- 文件：
  - `frontend-store/src/api/orders.ts`
  - `frontend-store/src/pages/Orders.tsx`
  - `frontend-store/src/pages/OrderDetail.tsx`
- 调整：
  - 为管理端订单 API 和页面补齐相同的订单实体类型。
  - 移除主要订单页中的 `any` 依赖。
  - 将错误提示统一改为按响应结构安全读取。
- 收益：
  - 管理端订单看板与详情页在后续迭代时更容易发现接口变更问题。
