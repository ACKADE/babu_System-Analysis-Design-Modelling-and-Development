### 3. 项目概述

我们计划开发一个基于双端架构的小型电商演示平台（Web应用程序）。主要功能包括：

**用户端：**
- 商品列表与分类筛选、搜索
- 商品详情与加入购物车
- 购物车管理（修改数量、移除商品）
- 结算与支付（模拟支付流程）
- 订单管理（取消/确认收货/售后）
- 注册/登录与个人中心（修改用户名/密码）
- 商品评价（订单完成后可评价）

**商店端：**
- 管理员登录（账号由种子数据/后台创建，不开放自助注册）
- 商品管理（添加、编辑、上架/下架、分类维护）
- 订单管理（发货、退货审批）
- 仪表盘统计
- 管理员个人中心

本项目不包含优惠券、直播、积分等复杂功能。

**技术栈：** Node.js + Express + TypeScript + Prisma + MySQL，使用 Docker Compose 实现一键部署；用户端与商店端前端均采用 React + Vite + TypeScript，接口集成 Swagger 文档。

### 5. 项目目标与范围

**面向人群（Who）：**
- 普通购物用户（如学生或有线上购物需求的人群）
- 商店运营者（管理商品与订单）
- 项目团队成员（开发与维护）

**项目内容（What）：**
一个小型电商双端平台，具备以下功能：
- 用户端：商品列表/详情、分类筛选、购物车管理、结算与模拟支付、订单管理（取消/售后/确认收货）、注册登录与个人中心、商品评价
- 商店端：管理员登录、商品/分类管理、订单管理（发货/退货审批）、仪表盘统计、管理员个人中心

**开发目的（Why）：**
为用户提供简洁的线上购物体验，为运营者提供简易的商品管理工具，同时锻炼全栈Web开发技能，实践前后端分离架构、RESTful API设计、数据库建模、Docker容器化部署及API文档自动化等现代工程实践。

**时间规划（When）：**
第3周启动开发，第12周前完成所有核心功能，第14周进行最终交付与演示答辩。

**运行环境（Where）：**
可通过浏览器本地访问，或部署至支持Docker的简易服务器；源代码托管于GitHub平台。

### 6. 方法论/开发方法

我们将采用**Agile（Scrum）敏捷开发方法**。

**理由：** 团队仅有4名成员，需求在开发过程中可能根据实际情况微调（如购物车交互细节、商店端功能优先级）。通过每两周一次的Sprint迭代，敏捷方法能够快速交付可运行的版本，并及时获得指导老师的反馈。

我们将使用GitHub Projects进行任务管理，每日进行线上站会同步进度，每周进行一次线下评审会议检查成果。

相比瀑布模型，敏捷方法更适用于课程项目中时间不确定性的情况，也有利于尽早发现并修复程序缺陷，这与我们控制风险的使命高度一致。
