# 小型电商平台完整实施计划（修订版）

## Phase 0：需求冻结与一致性校准
- 统一技术栈（双端 React + TS）
- 冻结核心状态机与接口契约
- 确认非功能需求（安全、稳定、可观测）

## Phase 1：后端核心能力
### 1.1 基础工程
- Express + TS 基础工程
- 全局错误处理、请求日志、健康检查

### 1.2 数据层
- Prisma Schema：User/Role/UserRole/Product/CartItem/Order/OrderItem/Review/PasswordResetToken
- 迁移脚本与种子数据

### 1.3 鉴权与权限
- 登录、刷新、登出、获取当前用户
- 重置密码令牌流程（申请 + 重置）
- RBAC（USER/ADMIN）中间件

### 1.4 交易能力
- 购物车增删改查
- 下单事务（库存扣减 + 订单快照 + 订单号生成）
- 取消订单、发货、确认收货、售后申请/审批

### 1.5 文档与测试
- Swagger 覆盖所有核心接口
- Jest + Supertest 覆盖认证与订单关键路径

## Phase 2：用户端实现
- 登录/注册/重置密码入口
- 商品列表/详情/搜索/分类
- 购物车、结算、支付成功页
- 订单列表、订单详情、取消、确认收货、售后申请
- 评价与个人中心

## Phase 3：商家端实现
- 管理员登录
- 商品管理（创建/编辑/上下架）
- 订单管理（发货）
- 售后审批（同意/拒绝）
- 仪表盘（商品数、订单数、销售额、待处理售后）

## Phase 4：质量与验收
- 回归测试：用户交易主链路 + 管理履约链路
- 安全检查：鉴权绕过、越权、重置密码滥用、上传攻击面
- 文档验收：需求、接口、页面行为保持一致

## 交付标准
- 用户端可独立完成下单到售后闭环
- 商家端可独立完成商品管理与订单履约闭环
- 核心接口均可在 Swagger 中复现
- 关键业务规则具备自动化测试覆盖
